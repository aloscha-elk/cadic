Scan Tool Data Definitions



SCAN TOOL DATA DEFINITIONS

The Body Control Module (BCM) scan tool data definitions are a brief description of each of the vehicle access related parameters displayed within the scan tool data displays for the BCM.

Door Lock Requestor: This parameter represents the door lock function responsible for a door lock activation.

Drvr. Door Unlock Cmd.: When the BCM sends a driver door unlock command to the DDS the parameter will be On.

Left Rear Door Ajar Sw.: This parameter indicates the current status of the door ajar switch input to the BCM.

Rear Doors Lock Cmd.: This parameter indicates the commanded state of the rear door lock relay control circuit.

Rear Doors Unlock Cmd.: This parameter indicates the commanded state of the rear door unlock relay control circuit.

Right Rear Door Ajar Sw.: This parameter indicates the current status of the door ajar switch input to the BCM. The Driver Door Switch (DDS) scan tool data definitions are a brief description of each of the vehicle access related parameters displayed within the scan tool data displays for the DDS.

Door Lock Actuator Cmd.: This parameter indicates the commanded state of the door lock actuator control circuits.

Door Key Cyl. Unlock Switch: This parameter indicates the current status of the door cylinder unlock switch.

Driver Door Ajar Sw.: This parameter indicates the current status of the door ajar switch input to the DDS.

Driver Door Open Sw.: This parameter indicates the current status of the door open switch input to the DDS. The Passenger Door Switch (PDS) scan tool data definitions are a brief description of each of the vehicle access related parameters displayed within the scan tool data displays for the PDS.

Door Lock Actuator Cmd.: This parameter indicates the commanded state of the door lock actuator control circuits.

Passenger Door Ajar Sw.: This parameter indicates the current status of the door ajar switch input to the PDS.

Passenger Door Open Sw.: This parameter indicates the current status of the door open switch input to the PDS. The Liftgate Module (LGM) scan tool data definitions are a brief description of each of the parameters displayed within the scan tool data displays for the LGM.

Battery Voltage Signal 2: The scan tool displays the battery voltage as seen at the liftgate module.

Hall Sensor Feed Status: The scan tool displays No Output/Short to Ground. Than scan tool will display any faults associated with the hall sensor while in motion.

Liftgate Handle Switch Status: The scan tool displays Active/Inactive. The scan tool the status of the liftgate handle switch and the liftgate close switch.

Liftgate Ajar Switch: This parameter represents the state of the ajar output to the rear wiper motor and BCM. Whenever the liftgate or liftglass is ajar the status will be Active.

Object Detection Sensor Status: The scan tool displays the voltage as seen through the side object sensors.

LGM Clutch Actuator Command: The scan tool displays Active/Inactive. The scan tool displays the commanded state of the liftgate clutch.

LGM Latch Motor Cinch Command: The scan tool displays Active/Inactive. The scan tool displays the commanded state of the cinching portion of the liftgate latch.

LGM Latch Pawl Switch: The scan tool displays Active/Inactive. The scan tool displays the status of the liftgate open switch.

LGM Latch Ratchet Switch: The scan tool displays Active/Inactive. The scan tool displays the status of the liftgate ajar switch.

LGM Motor Release Command: The scan tool displays Active/Inactive. The scan tool displays the commanded state of the release portion of the liftgate latch.

LGM Latch Sector Switch: The scan tool displays Active/Inactive. The scan tool displays the state of the liftgate cinch/release switch.

LGM Motor Closed Command: The scan tool displays Active/Inactive. The scan tool displays the commanded state of the liftgate motor during closing.

LGM Motor Open Command: The scan tool displays Active/Inactive. The scan tool displays the commanded state of the liftgate motor during opening.

LGM Open/Close Position Learned: The scan tool displays Yes/No. The scan tool displays the results of the liftgate calibration.

LGM Full Open Switch: The scan tool displays Active/Inactive. The scan tool displays the status of the liftgate full open switch. The Liftgate Module (LGM) scan tool open history data and closed history data list definitions are a brief description of each of the parameters displayed within the scan tool displays.

Crank Signal Present: If the LGM received the GMLAN engine cranking message the parameter status will be Yes.

Door Lock Status: This parameter indicates the status of the liftgate lock.

Hall Sensor Feed Status: This parameter reflects the status of the 5 volt reference to the hall switches. When the circuit has faulted the parameter state will be Short to Ground

Liftgate Handle Sw. Status: This parameter indicates the status of the liftgate handle and close switch signal to the LGM. When one of the switches becomes active during an open or close cycle the parameter status will be Active.

System Operation Status

System Voltage: If the battery supply circuit to the LGM remains within 9-16 volts during the open or close cycle the parameter status will be OK.

Temperature Out of Range: The LGM uses a combination of a GMLAN ambient air temperature message and an internal temperature sensor to determine the ambient air temperature, and if the calculated temperature was within -17°C (-1°F) to 80°C (176°F) the parameter status will be No.

Vehicle In Park: If the LGM received the GMLAN transmission in park message the parameter status will be Yes.
